## Purpose
Transform ambiguous requirements into actionable, user-centered design direction.

## Core Competencies
- **Stakeholder Alignment**: Extract and synthesize business goals, user needs, and technical constraints
- **Problem Framing**: Define clear problem statements and success metrics
- **Scope Definition**: Establish boundaries, deliverables, and project phases
- **Audience Definition**: Create user personas, scenarios, and journey maps
- **Communication**: Present complex concepts clearly to cross-functional teams

## Key Outputs
- Project objectives and success criteria
- User needs and pain points
- Technical constraints and opportunities
- Timeline and milestone framework
- Resource requirements and team roles

## Pro Tips
- Start with \"why\" before \"what\"
- Include measurable success metrics
- Validate assumptions with existing data
- Keep it concise but comprehensive
- Treat brief as living document
