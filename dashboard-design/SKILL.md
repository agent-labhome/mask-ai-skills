## Metadata
- **Category**: Data Visualization / UI/UX Design
- **Level**: Intermediate to Advanced
- **Prerequisites**: Basic data literacy, familiarity with data analysis tools, understanding of user experience principles
- **Related Skills**: Data Analysis, Information Architecture, Visual Design, UX Research
- **Tools**: Tableau, Power BI, Looker, Google Data Studio, Figma, Sketch, D3.js, Excel/Sheets
- **Time to Basic Proficiency**: 2-3 months with consistent practice
- **Time to Advanced Proficiency**: 1-2 years with project experience

## Practical Guide

### Core Principles

**1. Know Your Audience**
- Identify primary users (executives, analysts, operators)
- Understand their goals, decision-making needs, and technical proficiency
- Determine frequency of use (daily monitoring vs. quarterly review)

**2. Define Clear Objectives**
- What key decisions will this dashboard inform?
- What are the 3-5 most important metrics (KPIs)?
- What actions should users take after viewing the dashboard?

**3. Hierarchy and Layout**
- Place most important information top-left (following F-pattern reading)
- Group related metrics together
- Use consistent spacing and alignment (grid systems recommended)
- White space is functional, not wasted space

### Design Process

**Phase 1: Discovery**
```
1. Stakeholder interviews
2. User persona development
3. Data source audit
4. KPI definition workshop
```

**Phase 2: Information Architecture**
```
1. Create user flow diagrams
2. Develop metric hierarchy
3. Sketch low-fidelity wireframes
4. Validate with paper prototypes
```
