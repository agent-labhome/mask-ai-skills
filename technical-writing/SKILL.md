## Purpose
Communicate complex technical information clearly and accurately for specific audiences.

## Core Principles
- **Clarity**: Simple, precise language; avoid ambiguity
- **Accuracy**: Technically correct and verifiable information
- **Audience Awareness**: Tailor content to reader's knowledge level
- **Consistency**: Use standardized terminology and formatting
- **Completeness**: Provide all necessary information without gaps

## Key Deliverables
- Documentation (APIs, user guides, tutorials)
- Process documentation and SOPs
- Technical specifications and requirements
- Release notes and changelogs
- Knowledge base articles

## Essential Practices
- Start with outlines and structure before writing
- Use active voice and imperative mood for instructions
- Incorporate visual aids (diagrams, screenshots) when helpful
- Apply consistent formatting and style guides
- Review and test documentation for usability
