## Overview

Technical writing is the practice of creating clear, concise, and user-focused documentation that explains complex technical information. Unlike creative or academic writing, technical writing prioritizes clarity, accuracy, and usability above all else.

## Core Principles

### 1. **Audience Awareness**
- Identify your readers' technical level, goals, and pain points
- Write for the least experienced user who needs to understand the material
- Consider multiple audience segments when necessary

### 2. **Clarity and Precision**
- Use simple, direct language
- Define technical terms on first use
- Avoid ambiguity and vague phrasing
- Be specific and concrete

### 3. **Task-Oriented Approach**
- Focus on what users need to accomplish
- Structure content around user goals, not product features
- Provide actionable steps and practical examples

### 4. **Consistency**
- Maintain consistent terminology, formatting, and tone
- Follow established style guides and patterns
- Create and use templates for recurring documentation types

## Documentation Types

### **User-Facing Documentation**
- **Getting Started Guides:** Quick-start tutorials for new users
- **User Manuals:** Comprehensive guides covering all features
- **Tutorials:** Step-by-step learning paths with specific outcomes
- **How-to Guides:** Task-focused instructions for specific goals
- **Troubleshooting:** Problem-solution pairs for common issues

### **Developer Documentation**
- **API Documentation:** Endpoint descriptions, parameters, examples
- **SDK/ Library Documentation:** Code examples, installation guides
- **Architecture Documentation:** System design, data flow, components
- **Contributing Guides:** Onboarding for new developers

### **Process Documentation**
- **Standard Operating Procedures (SOPs):** Repeatable process instructions
- **Policies:** Rules and guidelines for organizational behavior
- **Knowledge Base Articles:** Reference material for internal teams
